DocuMind Local Config UI

Локальный dev tool для управления pipeline, типами документов, prompt templates и запусками.

Config: здесь редактируется config/config.jsonc. Это источник истины для pipeline, LLM-профилей, paths, debug и UI. Сохраняй изменения, затем запускай Config Doctor. Для реальных документов с ПДн production-профиль должен оставаться локальным/on-prem.
Field Mappings: понятный маппинг технических ключей, labels и подсказок для извлечения. Это отдельный config/field_mappings.json; его можно менять через UI без правки кода.
Required: если компонент упал, pipeline останавливается. Если required: false, ошибка попадёт в результат, но следующие шаги продолжатся.
Enabled: компонент вообще участвует в pipeline. Required: — критичность ошибки.
Doc Types: каждый файл в config/doc_types/*.json описывает один тип документа: aliases, признаки распознавания, поля первого pass и второй legal pass. Новый тип документа добавляется новым JSON-файлом — код менять не нужно.
Select doc type
Prompts: здесь редактируются prompt templates. Промпты динамически собираются из templates + выбранного doc type. Для первого pass — универсальное понимание документа. Для второго pass — широкий legal extraction без повторной отправки изображения при `imagePolicy: "session"`.
Select prompt
Run: Dry Run безопасен и не требует входных документов. Extract читает input/, пишет output/ и debug/. Сейчас активен local-lmstudio: http://127.0.0.1:1234/v1, API key не нужен.
